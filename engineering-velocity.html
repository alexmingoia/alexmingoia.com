Engineering velocity

By Alex Mingoia • 6 Jul 2023

The quickest way forward is knowing your next step. You can only move as fast as you have answers to “How do we ship?”

In other words, engineering velocity is knowing the way forward.

Many of the decisions between idea and release can be made once and leveraged repeatedly. Distilled they become standard operating procedures (SOPs).

Technical SOPs:

  • Linter and code formatter
  • Branching workflow
  • Test framework
  • Expected test coverage
  • Automated tests
  • Automated deployment
  • Standard UI library
  • Standard API schema (REST with flat resources, graphQL, etc.)
  • Data migration process
  • Data handling and PI checklist
  • Security checklist or linter

Organizational SOPs:

  • Definition of done for new features
  • Definition of ready for user stories, tickets, or project briefs
  • How and when new work is assigned
  • When and where progress is reported
  • How and when new work is reviewed

Do buttons need rounded corners? Maybe. I know if you have a standard UI library you don't have to revisit what a button looks like every new feature.

Do user stories need to be wireframed? Maybe. I know if you have a standard definition of ready you don't need ask the same questions every new feature.

"How do we ship?" is worth considering.
