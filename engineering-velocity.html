Engineering velocity

The quickest way forward is knowing your next step. You can only move as fast as you have answers to “How do we ship?”

In other words, engineering velocity is knowing the way forward.

Many of the decisions between idea and release can be made once and leveraged repeatedly. Distilled they become standard operating procedures (SOPs).

Technical SOPs:

  • Linter and code formatter
  • Branching workflow
  • Test framework
  • Expected test coverage
  • Automated tests
  • Automated deployment
  • UI library
  • data access patterns
  • API standard
  • Data migration process
  • Data handling and PI checklist
  • Security checklist or linter

Organizational SOPs:

  • Definition of done for new work
  • Definition of ready for user storiest and tickets
  • How work is assigned
  • How progress is reported
  • How work is accepted
  • Review checklist
  • Release checklist
